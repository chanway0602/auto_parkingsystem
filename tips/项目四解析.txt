4、数据库判断刷卡次数：

（1）函数解释：
	1）执行sqlite3中的命令--->sqlite3_exec()函数。
           int sqlite3_exec(
sqlite3 *,const char *sql, int (*callback)(void*,int,char**,char**), void *,char **errmsg);  

                   返回值:成功SQLITE_OK

                   参数: sqlite3 * --->你前面使用sqlite3_open()函数的第二个参数
                         sql	   --->你要执行的命令
                         int (*callback)(void*,int,char**,char**)--->函数指针――是个回调函数。
                                 当你执行select查询命令的时候,该参数就起作用。

                                 函数指针参数:  void *-->接收来自第四个参数传递过来的数据
                                                int --->表示表格中的列数
                                                char** --->每一行的信息
                                                char** --->字段名

	 2）注意重点重点:select命令查询的结果有几条,那么该指针就调用几次！
		该函数的返回值一定要写
                void *--->传递给函数指针指向的函数的参数
                errmsg --->存放出错信息

（2）利用函数的特性：
	特性：select命令查询的结果有几条,那么该函数指针就调用几次！
		也就是sqlite3_exec函数执行select查询命令的时候，表格中符合查询条件的结果有三条信息，那么sqlite3_exec函数中的第三个参数――函数指针就会被调用三次。
	
（3）那么函数指针指向的函数是自定义的，你可以打印数据表的信息，也是设置标志位。
	关键点：设置一个全局标志位flag=0，第一次刷卡前，数据表格是没有车主信息的；
					第一次刷卡后，数据表格是保存有车主信息的（前提是你添加信息进表格了）；sqlite3_exec函数执行select查询命令时，会把函数指针指向的自定义函数里的标志位置1（flag=1）。
					第二次刷卡后，就可以用这个全局标志位来判断车主是进库还是出库了。

（4）数据表格的扩展使用：
			1）记录的信息有：车主名字，电话，车牌号，当时入库时摄像头拍照（可直观看到车辆情况）。	

			2）车主停车位置的记录：刷卡停车在一号库，不可能出库时把别人二号库的特斯拉开走吧？

			3）车主是VIP，使用次数随年增加，那么数据表格里是否需要记录次数。
			   车主是临时卡，那么是否每次出库后删除数据，而且缴费金额也会不一样。
